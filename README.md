# 🚀 JVM Visualizer

> Interactive JVM Bytecode Analysis & Control Flow Visualization Platform

JVM Visualizer is a full-stack developer tool that transforms Java source code into an interactive representation of JVM execution. It dynamically compiles Java code in memory, extracts bytecode instructions, simulates operand stack behavior, generates Control Flow Graphs (CFGs), and visualizes execution flow through an intuitive web interface.

---

## ✨ Features

### 🔥 Dynamic Java Compilation

Compile Java source code entirely in memory using the Java Compiler API.

* No `.java` or `.class` files written to disk
* Fast in-memory compilation
* Runtime source code analysis

---

### ⚙️ Bytecode Extraction

Parse generated JVM bytecode using ASM.

#### Java Source

```java
public int add(int a, int b) {
    return a + b;
}
```

#### Generated Bytecode

```text
ILOAD 1
ILOAD 2
IADD
IRETURN
```

---

### 📚 Operand Stack Simulation

Visualize JVM stack transitions instruction by instruction.

| Instruction | Stack State |
| ----------- | ----------- |
| ILOAD 1     | [a]         |
| ILOAD 2     | [a, b]      |
| IADD        | [a+b]       |
| IRETURN     | []          |

---

### 🌳 Control Flow Graph (CFG) Generation

Generate CFGs directly from bytecode instructions.

Supports:

* Conditional branches
* Nested conditions
* Loops
* Unconditional jumps
* Multiple return paths

Example:

```text
          IF_ICMPGT
          /      \
      TRUE      FALSE
        |          |
     RETURN     RETURN
```

---

### 🎨 Interactive Graph Visualization

Built with React Flow.

Features:

* Zoom & Pan
* Interactive Nodes
* Directed Edges
* Loop Visualization
* Automatic Graph Layout
* Method-Level CFG Exploration

---

### 🧩 Method-Level Analysis

Analyze methods independently instead of merging the entire class.

```text
Demo
├── <init>()
├── main()
├── calculate()
└── multiply()
```

Each method contains:

* Bytecode Instructions
* Operand Stack Simulation
* Control Flow Graph

## 🧪 Example Workflow

### Input Java Code

```java
public int max(int a, int b) {

    if (a > b) {
        return a;
    }

    return b;
}
```

### Extracted Bytecode

```text
ILOAD 1
ILOAD 2
IF_ICMPLE 8
ILOAD 1
IRETURN
ILOAD 2
IRETURN
```

### Generated CFG

```text
            IF_ICMPLE
             /     \
         TRUE      FALSE
           |          |
      return b    return a
```

## ⭐ Why This Project?

Most Java developers write code without seeing how the JVM actually executes it.

JVM Visualizer bridges that gap by providing an interactive environment to explore:

* What bytecode gets generated
* How the operand stack changes
* How branches and loops are represented
* How control flows through a program
